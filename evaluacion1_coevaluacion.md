## PROYECTO A EVALUAR
Cada estudiante debe haber desarrollado un sitio web básico que incluya:
- Página principal (index.html) con estructura semántica HTML5
- Al menos 3 páginas internas enlazadas
- Estilos CSS externos (archivo styles.css)
- Diseño responsivo (mobile-first)
- Implementación de Flexbox o Grid
- Accesibilidad básica (alt en imágenes, contraste adecuado)

## INSTRUCCIONES PARA EL DOCENTE

### Proceso de aplicación:
1. Asignar aleatoriamente 3 evaluadores por estudiante
2. Proporcionar enlaces a los proyectos de cada estudiante
3. Establecer plazo claro de entrega
4. Revisar y consolidar evaluaciones
5. Proporcionar feedback consolidado a cada estudiante

### Criterios de validación:
- Descartar evaluaciones evidentemente sesgadas (todas 100 o todas 0)
- Promediar las 3 evaluaciones recibidas
- Considerar la calidad de los comentarios como parte de la evaluación

### Adaptaciones para estudiantes con necesidades especiales:
- Asignar evaluadores que hayan demostrado empatía y constructividad
- Proporcionar rúbrica simplificada si es necesario
- Permitir formato alternativo de evaluación si corresponde

### Integración con sistema MCP:
- Automatizar asignación de evaluadores
- Recolección automática de evaluaciones
- Generación de reportes consolidados
- Alertas para evaluaciones atrasadas o problemáticas
