## DESCRIPCIÓN DEL PROYECTO

Desarrolle un sitio web completo para una organización real o ficticia que cumpla con todos los requisitos técnicos y de diseño especificados a continuación.

### Requisitos técnicos obligatorios:
1. **Estructura HTML5 semántica** completa
2. **CSS externo** con organización modular
3. **Diseño responsivo** mobile-first (3 breakpoints mínimo)
4. **Implementación de Flexbox y/o Grid** demostrando comprensión
5. **Accesibilidad** básica implementada
6. **Optimización** de imágenes y recursos
7. **Validación** HTML y CSS sin errores críticos
8. **Hosting** en GitHub Pages o servicio similar

### Requisitos de contenido:
- Página principal (index.html) con presentación
- Al menos 3 páginas secundarias con contenido relevante
- Formulario de contacto funcional (puede ser estático)
- Galería de imágenes o productos
- Página "Acerca de" o similar

## DEFENSA TÉCNICA - CUESTIONARIO (30%)

### Instrucciones para la defensa:
Cada estudiante responderá preguntas específicas sobre su código durante 10-15 minutos. Las preguntas se centrarán en secciones específicas del código desarrollado.

### Ejemplo de preguntas (adaptadas al código de cada estudiante):

1. **Sobre estructura HTML:**
   - "Explique por qué usó la etiqueta `<section>` en lugar de `<div>` en la línea 45 del index.html"
   - "¿Cómo aseguró la accesibilidad en el formulario de contacto?"

2. **Sobre CSS y responsividad:**
   - "Explique su estrategia mobile-first y los breakpoints definidos"
   - "En el archivo styles.css, líneas 120-150, ¿por qué usó Flexbox en lugar de Grid para ese componente?"

3. **Sobre implementación específica:**
   - "En la galería de imágenes, ¿cómo implementó la optimización de carga?"
   - "Explique la función JavaScript en el archivo 'validation.js' líneas 30-50"

4. **Sobre decisiones de diseño:**
   - "¿Por qué eligió esa paleta de colores y cómo afecta la accesibilidad?"
   - "Justifique la estructura de archivos y carpetas que utilizó"

## INSTRUCCIONES PARA EL DOCENTE - PROCESO DE EVALUACIÓN

### 1. Preparación:
- Revisar todos los proyectos antes de las defensas
- Identificar secciones de código específicas para preguntar a cada estudiante
- Preparar preguntas personalizadas según el código de cada uno

### 2. Durante defensa:
- Hacer preguntas técnicas específicas
- Evaluar no solo respuesta correcta sino comprensión
- Tomar notas detalladas para feedback

### 3. Después de defensa:
- Consolidar puntajes proyecto + defensa
- Preparar feedback personalizado
- Identificar patrones comunes para retroalimentación grupal

### 4. Adaptaciones para necesidades especiales:
- **Tiempo extendido:** 20-25 minutos para defensa si es necesario
- **Formato alternativo:** Defensa escrita si la oral es problemática
- **Preguntas pre-definidas:** Entregar preguntas con anticipación si ayuda
- **Apoyo visual:** Permitir uso de diagramas o notas durante defensa

### 5. Integración con sistema MCP:
- Cargar rúbricas completas por estudiante
- Generar feedback automático basado en criterios
- Alertar sobre estudiantes en riesgo después de esta evaluación
- Actualizar perfil de aprendizaje de cada estudiante
